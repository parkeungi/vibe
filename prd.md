저와 함께 빌드업한 내용을 바탕으로, 개발자에게 바로 전달해도 손색없는 '바이브 스토어(Vibe Store)'의 최종 PRD(제품 요구사항 정의서)를 완성했습니다. 이 문서를 가이드 삼아 멋진 나만의 스토어를 런칭하시길 응원합니다!

📄 [PRD] 바이브 스토어 (Vibe Store) 제품 요구사항 정의서
1. 프로젝트 개요
프로젝트 이름: 바이브 스토어 (Vibe Store)

한 줄 설명: 나만의 지식 상품(전자책, 가이드북, 템플릿 등)을 편리하게 판매하고, 데이터 기반으로 매출을 관리하는 1인 비즈니스 커머스 플랫폼

유형: 웹 서비스 (PC 및 모바일 대응 가능한 반응형 웹)

난이도: 중급 (Intermediate)

이유: 소셜 로그인, PG 결제 시스템 연동, 보안 파일 다운로드, 관리자 매출 대시보드 시각화 기능 포함.

2. 사용자 시나리오
📱 구매자 (직장인, 취준생)
누가: 실무 역량을 키우거나 취업 성공을 위해 검증된 노하우(전자책, 템플릿 등)가 필요한 사람.

언제/왜: SNS, 블로그, 포트폴리오 링크를 통해 '바이브 스토어'에 유입되어 자신에게 필요한 상품을 발견했을 때.

핵심 경험: 복잡한 과정 없이 간편하게 결제하고, 마이페이지에서 즉시 상품 파일을 다운로드하여 자신의 업무나 공부에 바로 적용한다.

📊 판매자 (본인)
누가: 지식 상품을 통해 수익을 창출하고 비즈니스를 관리하려는 1인 창업자.

언제/왜: 새로운 지식 상품이 완성되어 등록해야 하거나, 매일 퇴근 후나 주말에 운영 성과를 점검하고 싶을 때.

핵심 경험: 관리자 대시보드에 접속해 새로운 상품을 등록/관리하고, 오늘 얼마나 팔렸는지 실시간 매출 추이를 한눈에 확인한다.

3. 핵심기능목록
📌 필수 기능 (MVP)
사용자(구매자) 기능

소셜 로그인 및 회원가입 (구글, 카카오 등)

상품 목록 조회 및 상세 페이지 (상품 설명, 가격, 평점)

결제 시스템 연동 (토스페이먼츠, 카카오페이, 신용카드 등)

마이페이지 (구매 내역 확인 및 보안 다운로드)

리뷰 및 별점 작성 기능

관리자(판매자) 기능

상품 관리 (상품 CRUD 및 실제 판매용 디지털 파일 업로드)

매출 대시보드 (기간별 총 매출액, 판매 건수, 실시간 매출 추이 그래프)

💡 선택 기능 (추후 고도화)
쿠폰 및 할인 코드 생성 기능

새로운 리뷰 등록 시 관리자 알림 기능 (카카오톡/이메일)

4. 화면구성
📱 구매자용 화면 (Client-side)
메인 / 상품 목록 페이지 (/)

지식 상품 그리드 카드 레이아웃. 고화질 썸네일, 상품 타이틀, 평점(★), 가격이 조화롭게 노출되는 구조.

상품 상세 페이지 (/products/:id)

상단 영역(Hero Section): 썸네일, 타이틀, 평점, 가격, [바로 구매하기] 버튼 배치.

하단 영역: 상품의 상세 소개글, 목차, 실제 구매자들의 리뷰 리스트.

로그인 페이지 (/login)

구글, 카카오 등 간편 소셜 로그인 버튼 중심의 심플한 화면.

마이페이지 (/mypage)

내가 구매한 상품 목록 확인 및 상태가 [결제 완료]인 경우 [파일 다운로드] 버튼 제공.

📊 판매자용 화면 (Admin-side)
관리자 로그인 페이지 (/admin/login)

일반 유저와 격리된 관리자 전용 인증 화면.

대시보드 메인 (/admin/dashboard)

이번 달 총 매출, 오늘 판매 건수, 실시간 매출 추이를 시각화한 그래프 및 카드 UI.

상품 관리 목록 페이지 (/admin/products)

등록된 상품 리스트 조회, 수정/삭제 링크 및 [상품 등록] 버튼.

상품 등록 및 수정 페이지 (/admin/products/new)

상품 정보 입력 폼 및 판매용 파일(PDF, Zip 등) 업로드 영역.

5. 상세기능명세
💳 1. 결제 및 파일 다운로드 로직
주문 데이터가 [결제 완료] 상태일 때만 마이페이지에서 [다운로드] 버튼이 활성화됩니다.

파일 주소(URL) 직접 입력 등을 통한 불법 다운로드를 방지하기 위해, 서버 세션에서 실제 결제 여부를 검증한 뒤 다운로드 스트림을 인가합니다.

✉️ 2. 알림 시스템 (이메일)
PG 결제창을 통해 결제 완료 승인이 떨어지는 즉시 구매자 계정으로 '구매 확인 이메일'을 자동 발송합니다.

메일에는 상품명, 결제 금액, 결제 일시, 마이페이지 바로가기 링크만 깔끔하게 포함합니다.

⭐ 3. 리뷰 및 평점 권한 시스템
리뷰 작성 컴포넌트는 유저 ID와 상품 ID를 대조하여, 실제 구매 내역이 존재하는 유저에게만 노출됩니다.

비구매자에게는 안내 문구를 노출하며, 어뷰징 방지를 위해 주문 건당 1회만 작성 가능하도록 제한합니다.

6. 디자인가이드
전체 무드: 신뢰감을 주는 미니멀리즘 (토스, 노션 스타일의 깔끔하고 가독성 높은 UI)

컬러 시스템 (Color Palette)

배경색: Pure White (#FFFFFF) & Light Gray (#F9FAFB)

텍스트: Charcoal Black (#111827)로 깊이감 있는 가독성 확보

포인트 컬러: Sleek Indigo Blue (#2563EB) -> 버튼 및 강조 영역

별점 컬러: Warm Amber (#FBBF24) -> 평점 별 모양(★)에만 적용하여 가시성 극대화

컴포넌트 가이드

상품 카드: 썸네일 이미지는 부드러운 라운딩(Border-radius: 12px) 처리.

계층 구조: 타이틀 ➡️ [★ 4.9 (12) · 카테고리] ➡️ 가장 굵은 가격 폰트 순으로 배치하여 시각적 안정감 유도.

7. 기술스택
프레임워크 (Full-stack): Next.js (React 기반, 프론트/백엔드 API 통합 개발)

스타일링 (UI): Tailwind CSS (빠른 스타일링 및 미니멀 UI 구현)

데이터베이스 & 스토리지: Supabase (회원 정보, 상품 DB 및 디지털 파일 저장용 Storage 활용)

결제 시스템 (PG): 포트원 (Portone) API (토스페이먼츠, 카카오페이 등 연동)

인프라 & 배포: Vercel (Next.js 최적화 호스팅 및 GitHub 연동을 통한 자동 배포)

8. 제약사항
초기 구축 비용 최적화: 초기 트래픽 단계를 감안하여 Vercel, Supabase의 Free Tier(무료 플랜) 범위 내에서 구동되도록 아키텍처를 설계합니다.

파일 업로드 제한: Supabase Free Tier 스토리지 용량을 고려하여, 등록 가능한 개별 지식 상품 파일(PDF, ZIP 등)의 최대 크기는 단일 파일 기준 50MB 이하로 제한합니다.

보안: 관리자 페이지(/admin/*)는 인가되지 않은 일반 사용자의 접근을 미들웨어(Middleware) 레벨에서 원천 차단해야 합니다.
